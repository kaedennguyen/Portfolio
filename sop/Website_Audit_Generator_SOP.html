1. Purpose & Scope

This SOP documents how to install, configure, operate, maintain, and troubleshoot the Website Audit Generator — a WordPress plugin that lets a site visitor enter a URL and receive an automated audit scored across five categories: SEO, AEO, GEO, Content, and Design & UX.

This document is intended for the plugin administrator (site owner) responsible for installation, API key management, and ongoing monitoring. It is not an end-user manual for site visitors — visitor-facing instructions are built into the tool itself as an on-page intro message.

1.1 In Scope

  • Installing and configuring the plugin on a WordPress site
  • Publishing the audit tool via shortcode
  • Monitoring API usage, cost, and daily limits
  • Troubleshooting common failure modes (timeouts, rate limits, blocked requests)
  • Routine maintenance and update procedures

1.2 Out of Scope

  • General WordPress hosting/server administration
  • Visual/screenshot-based design critique (not implemented — see Section 3.2 of the plugin's design)
  • Auditing JavaScript-rendered (client-side-only) websites with full accuracy (see Section 8.4)

2. System Overview

The plugin is fully self-contained within WordPress — there is no separate backend server. It calls two external services at audit time:

  • Google PageSpeed Insights API — free, provides performance/accessibility/SEO signals
  • Anthropic Claude API — paid per-use, provides qualitative judgment (AEO, GEO, Content, Design commentary)

2.1 Architecture Summary

When a visitor submits a URL:

  1. The plugin fetches and parses the target page's raw HTML server-side (PHP DOMDocument) — no headless browser rendering is performed.
  2. It builds four independent Claude requests (AEO, GEO, Content, Design) plus one PageSpeed request.
  3. All requests fire concurrently via curl_multi, not sequentially, to stay within PHP execution time limits.
  4. Each category analyzer (SEO, AEO, GEO, Content, Design) combines deterministic structural scoring with the Claude qualitative result.
  5. A weighted overall score (0–100, graded A–F) is computed and displayed, with results cached to reduce repeat API cost.

2.3 Scoring Categories & Weights

CategoryWeightWhat it measures
SEO25%Title/meta tags, headings, schema, links, PageSpeed SEO + Performance scores
AEO20%Answer Engine Optimization — readiness for featured snippets/voice search (FAQ schema, question headings)
GEO20%Generative Engine Optimization — likelihood of being cited by AI systems like ChatGPT/Perplexity (freshness, authorship, authority)
Content20%Readability (Flesch), word count, qualitative content-gap analysis
Design & UX15%Structural HTML analysis + PageSpeed accessibility/best-practices (no visual/screenshot review)
NOTE: AEO and GEO are related but distinct: AEO asks whether content is structured to be pulled into a direct-answer box; GEO asks whether a generative AI would cite the page as a source when synthesizing an answer.

3. Prerequisites

3.1 Requirements

  • WordPress 5.8 or later
  • PHP 7.4 or later, with the curl extension enabled (curl_multi is used for concurrent API calls; the plugin falls back to sequential calls automatically if unavailable)
  • A hosting plan that allows outbound HTTP requests to googleapis.com and api.anthropic.com

3.2 Required Accounts / Keys

KeyRequired?Where to get it
Claude API keyRequiredconsole.anthropic.com — used for AEO, GEO, Content, and Design qualitative analysis
Google PageSpeed API keyOptional (recommended)console.cloud.google.com — enable the "PageSpeed Insights API"; raises rate limits above the free unauthenticated tier

4. Installation Procedure

  1. Download the plugin zip file (website-audit-generator.zip).
  2. In WordPress admin, go to Plugins → Add New → Upload Plugin.
  3. Select the zip file and click Install Now.
  4. Click Activate once installation completes.
IF THIS FAILS: Confirm the PHP version meets the 7.4+ requirement (Tools → Site Health → Info → Server in WordPress admin).

5. Configuration Procedure

5.1 Adding API Keys

  1. Go to Settings → Audit Generator in the WordPress admin sidebar.
  2. Paste the Claude API key into the "Claude API Key" field. This is required — without it, AEO/GEO/Content/Design scores will fall back to structural-signal-only scoring and display a warning.
  3. (Optional) Paste the Google PageSpeed API key into the "Google PageSpeed API Key" field.
  4. Confirm the "Claude Model" field reads claude-sonnet-5 (the current recommended default as of this document's last update).
  5. Set the "Daily Audit Limit" — default 25/day site-wide. Adjust based on expected traffic and acceptable API spend.
  6. Set "Result Cache (hours)" — default 24. Repeat audits of the same URL within this window are served from cache at no additional API cost.
  7. Click Save Changes.

5.2 Publishing the Audit Tool

  1. Create or edit the WordPress page/post where the tool should appear.
  2. Add the shortcode: [website_audit_generator]
  3. Publish or update the page.

The published page will show an intro message explaining the five scoring categories, a URL input field, and (after submission) the full audit results with the contact email displayed at the bottom.

6. Standard Operating Procedure — Running an Audit

This describes the expected end-to-end flow for a site visitor using the published tool.

  1. Visitor lands on the page containing the [website_audit_generator] shortcode and reads the intro message.
  2. Visitor enters a URL (must begin with http:// or https://) and clicks "Audit My Site".
  3. The tool shows a loading spinner with the message "Running your audit — this can take 15–30 seconds…"
  4. On success, results display: an overall score/grade circle, followed by five collapsible category cards (SEO, AEO, GEO, Content, Design & UX), each expandable to show individual findings marked as good / warning / issue / improvement / summary.
  5. A contact line at the bottom invites further inquiries to mediavinescorp@gmail.com.
NOTE: If the same URL was audited within the cache window (default 24 hours), the cached result is served instantly and labeled "cached result" in the UI — no new API calls are made.

7. Maintenance Procedures

7.1 Routine Monitoring (Recommended: Weekly)

  • Check Anthropic Console (console.anthropic.com) for API usage/spend.
  • Check Google Cloud Console for PageSpeed API quota usage (free tier: 25,000 requests/day).
  • Spot-check a known URL to confirm the tool still returns results end-to-end.

7.2 Cost Control Checklist

  • Daily Audit Limit is set to a sensible ceiling for expected traffic.
  • Result Cache is enabled (non-zero) to avoid paying for repeat audits of the same URL.
  • Claude model is set to the most current cost-effective model (check Anthropic's pricing page periodically, as model names/pricing change over time).

7.3 Updating the Plugin

  1. Deactivate the current version in Plugins → Installed Plugins (settings/API keys are preserved in the database and are not lost on deactivation).
  2. Delete the old plugin files or upload the new zip to overwrite them via SFTP/hosting file manager.
  3. Reactivate the plugin.
  4. Re-verify Settings → Audit Generator still shows the correct API keys and run a test audit.

8. Troubleshooting Guide

8.1 Audit hangs / never returns a result

Cause: Historically, PageSpeed + Claude calls ran sequentially and could combine to exceed 2+ minutes, surpassing most hosts' PHP execution time limit.

Fix implemented (v1.1+): All four external API calls now run concurrently via curl_multi with a hard 60-second batch ceiling, and the AJAX handler sets a 90-second execution time budget. If this still occurs:

  • Confirm the host's max_execution_time is at least 90 seconds (check with hosting support or php.ini).
  • Confirm curl_multi is enabled on the server (rare hosts disable it — the plugin falls back to sequential calls automatically, which will be slower).

8.2 HTTP 429 errors

This can originate from three different places — the error message now specifies which:

SourceMeaningFix
Target websiteThe site being audited is rate-limiting or blocking automated requests (bot protection/WAF)Not fixable from the plugin side beyond looking browser-like; retry later or on a different page of the same site
Google PageSpeed APIUnauthenticated quota exceededAdd a PageSpeed API key under Settings → Audit Generator
Claude APIAccount rate limit reachedWait and retry, or review rate limits at console.anthropic.com

8.3 "Invalid URL" or fetch failures

  • Confirm the URL includes http:// or https:// — required by validation.
  • A 403 response means the target site's bot protection blocked the request outright.
  • Check the "Could not fetch the target URL" error text for the specific underlying reason — it is surfaced directly from the target server's response.

8.4 Inaccurate word count / content score on a specific site

This is expected behavior, not a bug, for JavaScript-rendered (client-side-only) sites — e.g. a bare React/Vue shell with no server-rendered HTML. The plugin reads only the raw HTML response; it does not execute JavaScript. Content-heavy static or server-rendered pages (most WordPress sites) are unaffected.

8.5 No Claude-based commentary appears / falls back to structural-only scoring

  • Confirm a Claude API key is entered under Settings → Audit Generator.
  • Check for an on-page warning finding stating the specific Claude API error returned.

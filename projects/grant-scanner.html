How fit scores work

Each score (0–100) combines four checks: mission/program alignment (does the funder's stated priority actually match a program this org runs, not just the word "hunger"), geographic eligibility (is the org's location inside the funder's actual service area), award-size fit (is the grant amount meaningful but realistic for a $2.5M-budget organization to absorb and report on), and deadline feasibility (is there enough runway to actually assemble a competitive application). A grant can sound like a great thematic match and still score Low if it's geographically restricted or structurally built for a different kind of organization — that's deliberate, not a bug. Two of the eight results below score Low for exactly that reason, which is worth noticing.

Next features I'd build with more time

Deadline auto-tracking

Sync each grant's deadline to a shared calendar and send staff a reminder at 60/30/14 days out, so nothing quietly expires between board meetings.

First-pass LOI drafting

For any grant marked High or Medium, generate a draft Letter of Inquiry pre-filled with the org's mission and the specific program that matches the funder's priorities — a starting point, not a final draft.

Outcome tracking loop

Let staff mark grants as Applied / Awarded / Declined, and feed those outcomes back into the fit-scoring logic so it gets sharper about this org's actual win patterns over time.
